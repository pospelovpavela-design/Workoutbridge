# Workoutbridge

Automatically sync Strava workouts to Nike Running Club via Garmin Connect.

```
Strava ──[Workoutbridge]──► Garmin Connect ──► Nike Run Club
                                               (Garmin's native sync)
```

## How it works

1. User connects Strava + Garmin accounts in the app (OAuth)
2. User links Nike Run Club inside Garmin's own Connected Apps (one-time setup)
3. Strava fires a webhook when a new activity is recorded
4. Workoutbridge converts it to TCX and uploads to Garmin
5. Garmin automatically pushes it to Nike Run Club

## Stack

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 15 (App Router), React 19, Tailwind CSS 4 |
| Auth | NextAuth.js v5 (credentials + JWT) |
| Database | PostgreSQL via Drizzle ORM (Neon recommended) |
| Queue | BullMQ + Redis for local dev; Vercel Cron for prod |
| Hosting | Vercel |

## Local Development

### Prerequisites
- Node.js 20+
- PostgreSQL database (or Neon free tier)
- Redis (local or Upstash free tier)
- Strava API app
- Garmin developer account

### Setup

```bash
# 1. Install dependencies
npm install

# 2. Copy env vars
cp .env.example .env.local
# Fill in all values in .env.local

# 3. Push schema to database
npm run db:push

# 4. Start the Next.js dev server
npm run dev

# 5. In a second terminal, start the job worker
npm run worker
```

App runs at `http://localhost:3000`.

### Strava Webhook (local)

Strava needs to reach your local server. Use [ngrok](https://ngrok.com):

```bash
ngrok http 3000
# Copy the https URL, e.g. https://abc123.ngrok.io
```

Then register the webhook:

```bash
curl -X POST https://www.strava.com/api/v3/push_subscriptions \
  -d client_id=$STRAVA_CLIENT_ID \
  -d client_secret=$STRAVA_CLIENT_SECRET \
  -d callback_url=https://abc123.ngrok.io/api/webhooks/strava \
  -d verify_token=$STRAVA_WEBHOOK_VERIFY_TOKEN
```

## Production Deployment (Vercel)

### 1. Create external services

| Service | Purpose | Free tier |
|---------|---------|-----------|
| [Neon](https://neon.tech) | PostgreSQL | yes |
| [Upstash](https://upstash.com) | Redis (optional, for local worker) | yes |
| [Resend](https://resend.com) | Email notifications | yes, 3k/mo |

### 2. Deploy to Vercel

```bash
npm i -g vercel
vercel
```

### 3. Set environment variables

Copy all values from `.env.example` into:  
**Vercel → Project → Settings → Environment Variables**

`CRON_SECRET` is auto-generated by Vercel — copy it from the dashboard.

### 4. Push DB schema

```bash
DATABASE_URL=<your-neon-url> npm run db:push
```

### 5. Register Strava webhook (production)

```bash
curl -X POST https://www.strava.com/api/v3/push_subscriptions \
  -d client_id=$STRAVA_CLIENT_ID \
  -d client_secret=$STRAVA_CLIENT_SECRET \
  -d callback_url=https://yourdomain.vercel.app/api/webhooks/strava \
  -d verify_token=$STRAVA_WEBHOOK_VERIFY_TOKEN
```

### 6. Link Nike Run Club in Garmin

Each user does this once in the Garmin Connect app:  
**More → Connected Apps → Nike Run Club → Connect**

## Architecture Notes

### Job processing (production)
- Webhook → inserts `sync_events` row (status: `pending`)
- Vercel Cron calls `/api/cron/process-jobs` every minute
- Picks up to 5 pending jobs with `SELECT FOR UPDATE SKIP LOCKED`
- On final failure (3 attempts) → email notification via Resend

### Job processing (local dev)
- Same webhook → DB insert, plus enqueues to BullMQ
- `npm run worker` runs a BullMQ worker for immediate processing

### Garmin → Nike
No code involved. Once the user connects Nike Run Club in Garmin's Connected Apps,
Garmin handles the push automatically for every new activity.
